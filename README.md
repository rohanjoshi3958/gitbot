# Git Commit Message Button (VS Code Extension)

Generate a detailed commit message with one click and auto-fill the Source Control input.

## Features

- Adds `Generate Commit Message` command.
- Adds a status bar button: `Commit Msg`.
- Adds a Source Control title bar button.
- Reads git changes (staged, unstaged, untracked) and drafts a detailed message.

## Run locally

1. Open this extension folder in VS Code.
2. Press `F5` to launch an Extension Development Host.
3. In the new window, open any git repository.
4. Make changes in that repo.
5. Click `Commit Msg` (status bar) or run `Generate Commit Message`.
6. The message is inserted into Source Control commit input.

## Settings

- `gitCommitMessageButton.maxDiffBytes`: max diff bytes to read.
- `gitCommitMessageButton.includeBody`: include bullet-point body.
- `gitCommitMessageButton.useAnthropic`: enable Anthropic generation.
- `gitCommitMessageButton.anthropicApiKey`: your Anthropic API key.
- `gitCommitMessageButton.anthropicModel`: model id (default `claude-3-5-sonnet-latest`).
- `gitCommitMessageButton.anthropicMaxTokens`: max output tokens.
- `gitCommitMessageButton.maxFileContentChars`: max chars per previous/current file snapshot sent to Claude.

## Anthropic setup

1. Open VS Code settings JSON.
2. Add:

```json
{
  "gitCommitMessageButton.useAnthropic": true,
  "gitCommitMessageButton.anthropicApiKey": "YOUR_ANTHROPIC_KEY",
  "gitCommitMessageButton.anthropicModel": "claude-3-5-sonnet-latest"
}
```

If Anthropic is unavailable or misconfigured, the extension automatically falls
back to local message generation.

When you run the command, the notification now shows whether the message was
generated by `claude` or `local`.

Claude mode now includes per-file `previous (HEAD)` vs `current` snapshots, so
the generated message can explicitly describe what changed and what that change
does.

## Publish to Marketplace

1. Create a Marketplace publisher and make sure `publisher` in `package.json`
   matches it exactly.
2. Create an Azure DevOps PAT with Marketplace publish permissions.
3. Login once:

```bash
vsce login <your-publisher-name>
```

4. Package and publish:

```bash
npm install
npm run package
npm run publish:patch
```

You can also publish a `.vsix` without releasing to Marketplace:

```bash
npm install
npm run package
```

## Notes before first public release

- Update `publisher` in `package.json` from `local-dev` to your real publisher id.
- Consider adding `repository`, `homepage`, and `bugs` fields in `package.json` once you have a public repo.
